1. 의사 결정 나무(Decision Tree)¶

  • 의사결정나무는 매우 쉽고, 데이터의 스케일링이나 정규화 등의 사전가공 영향이 적음
  • 하지만 예측성능을 향상시키기 위해 복잡한 규칙구조를 가져야 하고 이로 인한 과적합이 발생하여 예측성능 떨어질수도 있음
  • 트리의 depth가 깊어질수록 의사결정나무의 예측성능은 저하될 수 있음
  • 이러한 특성이 앙상블에는 적합--> 앙상블은 여러개의 약한 학습기를 결합하여 성능을 향상시키기 때문

4) visualize_boundary() - Classifier의 Decision Boundary를 시각화 하는 함수¶

  • 좌표를 찍는 scatter()과 구분영역을 표시하는 contourf()로 구성
In [ ]:
def visualize_boundary(model, X, y):
    fig,ax = plt.subplots()
    
    # 1. 학습 데이타 scatter plot으로 나타내기
    ax.scatter(x=X[:, 0], y=X[:, 1], c=y, s=25, cmap='rainbow', edgecolor='k',
               clim=(y.min(), y.max()), zorder=3)
    xlim_start , xlim_end = ax.get_xlim()
    ylim_start , ylim_end = ax.get_ylim()
    
    # 2. 파라미터로 들어온 model과 training 데이타로 학습 (fit) 
    model.fit(X, y)
    
    # 3.meshgrid 형태인 모든 좌표값으로 예측 수행  (predict) 
    # X,Y = meshgrid(x,y)는  x 및 y에 포함된 좌표를 2차원 그리드 좌표로 반환
    xx, yy = np.meshgrid(np.linspace(xlim_start,xlim_end, num=200),
                         np.linspace(ylim_start,ylim_end, num=200))
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)
    
    # 4.contourf()(채워진 등고선 그리기) 를 이용하여 class boundary 를 visualization 수행. 
    n_classes = len(np.unique(y))
    contours = ax.contourf(xx, yy, Z, alpha=0.3,
                           levels=np.arange(n_classes + 1) - 0.5,
                           cmap='rainbow', clim=(y.min(), y.max()),
                           zorder=1)

7) 결론! 의사결정나무의 엄격한 학습규칙으로 인한 과적합이 확인됨!¶

  • 일부 이상치 데이터까지 분류하기 위해 분할이 많이 일어나서 결정 기준 경계가 매우 많아짐
  • 의사결정나무의 기본 설정은 말단 노드에서 데이터가 모두 균일해야 하기 때문
  • 이렇게 복잡한 모델은 학습데이터 세트의 특성과 약간만 다른 형태의 데이터 세트가 주어지면 예측 정확도가 떨어짐
