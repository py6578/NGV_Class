1. Data 불러오기 (train.csv)¶

1) 먼저, DataField 확인 이해¶

  • [년월일시 - datetime관련]

datetime - hourly date + timestamp (년-월-일-시-분-초 형태)

  • [구분자(category)관련]

season – 1 = spring, 2 = summer, 3 = fall, 4 = winter ==> 1-3월, 4-6월, 7-9월, 10-12월임. 즉 1사분기, 2사분기, 3사분기, 4사분기임

holiday – whether the day is considered a holiday (공휴일이면 1, 아니면 0)

workingday - whether the day is neither a weekend nor holiday (근무일이면 1, 아니면 0)

  • [날씨, 기후 관련]

weather (날씨)

1: Clear, Few clouds, Partly cloudy, Partly cloudy

2: Mist + Cloudy, Mist + Broken clouds, Mist + Few clouds, Mist

3: Light Snow, Light Rain + Thunderstorm + Scattered clouds, Light Rain + Scattered clouds

4: Heavy Rain + Ice Pallets + Thunderstorm + Mist, Snow + Fog

temp - temperature in Celsius (온도)

atemp - "feels like" temperature in Celsius (체감온도)

humidity - relative humidity (습도)

windspeed - wind speed (풍속)

  • [자전거 대여량]

casual - number of non-registered user rentals initiated (비회원의 대여량)

registered - number of registered user rentals initiated (회원의 대여량)

count - number of total rentals (총 대여량)

6) 다중공선성 문제 (multicollinearity problem)¶

  • 회귀분석에서 독립변수들 간에 강한 상관관계가 나타나는 문제로 적절한 회귀분석을 위해 해결해야 함
  • 1) 하나의 변수로 치환
  • 2) 상관관계가 높은 독립변수 중 하나 혹은 일부를 제거
  • drop( 컬럼명, axis=1(컬럼삭제일때), inplace=True(삭제후 원본데이터에 덮어쓸때)

  • 1. 차원축소: temp와 atemp (2차원)를 하나의 변수 (1차원)로 변경함
  • 주성분 분석(Principal component analysis; PCA)
  • 고차원의 데이터를 저차원의 데이터로 환원시키는 기법
  • 직교 변환을 사용하여 서로 연관 가능성이 있는 고차원 공간의 표본들을 선형 연관성이 없는 저차원 공간(주성분)의 표본으로 변환

[참고] 로그를 취하여 분포를 변환시키는 이유¶

  • 0<x<1 --> (-무한대<y<0)으로 기울기가 가파름

  • 따라서 0에 가깝게 모여있는 x값들에 로그를 취하면 y값은 매우 큰 범위로 벌어짐

  • 반면 x>1 --> y값이 크게 차이나지 않음

  • 즉, 1 이상의 넓은 범위의 x는 로그를 취하면 비교적 작은 구간내의 y값으로 변환됨
  • 결국, 밀집되어 있는 데이터는 퍼지게, 퍼진 데이터는 밀집되게 하는 효과
  • 따라서 한쪽으로 몰려있는 데이터에 로그를 취하면 넓게 퍼지게 함으로써 분포의 왜곡을 완화하는 효과가 발생함
